# 29. Annexes : style de code

Le style de code n’est pas une religion. Il sert à rendre le code lisible et à réduire l’effort cognitif.

## Un style pragmatique

Une ligne, une idée. Des noms explicites. Des fonctions courtes.

## Le style comme outil

Le style n’est pas là pour « faire joli ». Il est là pour rendre le code plus facile à relire, à comprendre, et à maintenir.
