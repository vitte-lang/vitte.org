# 30. Annexes : FAQ

## Pourquoi Vitte ?

Parce qu’un code bas niveau peut être clair, stable, et agréable à relire.

## Vitte est‑il un langage « safe » ?

Vitte privilégie l’explicite. Il ne promet pas une sécurité magique, mais il vous donne les outils pour écrire un code robuste.

## Comment contribuer ?

Commencez par un test, un bug report, ou un petit patch documenté.

## Où trouver la doc ?

Le livre est votre guide principal, et les diagnostics du compilateur sont un second guide. Si quelque chose vous semble flou, commencez par réduire le cas minimal.
